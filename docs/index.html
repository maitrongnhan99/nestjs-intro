NestJS Learning Project

This repository contains a learning project following the "NestJS Masterclass - NodeJS Framework Backend Development" course on Udemy.

Project Description

This project is a hands-on implementation of concepts taught in the NestJS Masterclass course. NestJS is a progressive Node.js framework for building efficient, reliable, and scalable server-side applications.

Prerequisites

  • Node.js (v18.x or higher recommended)
  • npm or yarn

Installation

Example :
# Clone the repository
git clone <repository-url>
cd nestjs-intro

# Install dependencies
npm install
# or with yarn
yarn install

Running the Application

Example :
# Development mode
npm run start
# or
yarn start

# Watch mode (recommended for development)
npm run start:dev
# or
yarn start:dev

# Production mode
npm run start:prod
# or
yarn start:prod

Testing

Example :
# Unit tests
npm run test
# or
yarn test

# E2E tests
npm run test:e2e
# or
yarn test:e2e

# Test coverage
npm run test:cov
# or
yarn test:cov

Project Structure

The project follows the standard NestJS architecture:

  • src/ - Contains the source code
    • main.ts - The entry point of the application
    • app.module.ts - The root module of the application
    • Other modules, controllers, and services

Features

This project demonstrates various NestJS features including:

  • Dependency Injection
  • Module architecture
  • Controllers and Routing
  • Services
  • Middleware
  • Validation
  • Error handling
  • And more...

Contributing

Contributions are welcome! Here's how you can contribute to this project:

  1. Fork the repository

    Example :
    git clone https://github.com/yourusername/project-name.git
cd project-name

  2. Install dependencies

    Example :
    npm install
# or
yarn install

  3. Create a branch for your feature

    Example :
    git checkout -b feature/your-feature-name

  4. Make your changes

    • Follow the existing code style
    • Add tests for new functionality
    • Ensure all tests pass with npm run test

  5. Commit your changes

    Example :
    git commit -m "feat: add your feature description"

  6. Push to your branch

    Example :
    git push origin feature/your-feature-name

  7. Create a Pull Request

    • Go to the original repository
    • Click on "New Pull Request"
    • Select your branch and provide a description of your changes
